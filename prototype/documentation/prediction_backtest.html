1 · Why backtest at all

The prediction idea is guilt by association: a gene's function can be inferred from the company it keeps — the genes it co-behaves with. But two worries could make that worthless:

  • Is there any signal? Maybe co-behaving genes don't actually share function, and we'd just be generating noise dressed as insight.
  • Is it circular? If we predict a gene's function from data that was itself built from function annotations, we'd just be laundering known annotations back out and calling it a "prediction".

A backtest answers both with numbers, on genes whose functions we already know.

2 · The idea: an open-book exam we already have the answers to

Take a batch of genes whose functions are already well documented. Hide the answers, let the method guess each gene's functions using only its neighbours, then grade against the hidden answer key. If it scores far above random on genes we know, we can trust it on genes we don't know — which is the actual product.

Key property — no cheating. When predicting a gene's functions, the engine is never allowed to look at that gene's own annotations. It uses only the neighbours' annotations. The gene's real functions are used only to grade the result afterwards.

3 · The method, step by step

  1. Pick a target gene & seal its answer key. Its true GO Biological-Process functions are set aside — the predictor cannot see them.
  2. Gather neighbours. From one association layer (e.g. STRING), take the target's top-25 partners — other genes it associates with — each with a confidence weight.
  3. Let neighbours vote. For every function, add up the weights of the neighbours annotated with it. A function backed by many high-confidence neighbours scores high.
  4. Rank → predictions. Sort functions by score. This ranked list is the prediction — assembled entirely from neighbours.
  5. Unseal & grade. Reveal the target's real functions and check where they land in the ranking.

5 · The two metrics

hit@k — does a real function make the top-k?

For each gene, ask: is at least one of its sealed real functions among the top-k predictions? (yes = 1, no = 0). Averaged over all test genes. hit@10 = 0.61 means for 61% of genes, a real function landed in the top-10 — the metric that maps directly to the product ("here are the top predictions; how often is a real one in there?").

AUROC — is the whole ranking right?

Across all 7,522 candidate functions, rank by score and ask whether the gene's true functions outrank its non-functions (the probability a true function beats a random non-function). It rewards partial hits ranked 11th, 12th… too, so it measures overall ranking quality. 0.5 = no signal; 1.0 = perfect.

The null baseline is the whole point. The same test is run with 25 random genes as "neighbours". Random neighbours only surface common, high-frequency functions, so a true function reaches the top-10 just 2% of the time. Any layer that beats this floor carries real signal — and it also proves the score isn't just "common terms win".

7 · The circularity test

STRING scores best — but part of a STRING edge comes from a database channel (curated pathways) and a textmining channel (literature co-mention), both derived from the same knowledge GO annotations come from. Predicting GO from those channels would be partly circular. So we re-ran STRING using only the annotation-independent channels — experimental, co-expression, and genomic context — recombined the way STRING does.

Circularity is not the driver. Removing both annotation-derived channels barely moved the result (hit@10 0.68 → 0.61; AUROC 0.815 → 0.791). About 90% of STRING's predictive power survives using only independent experimental evidence. And co-essentiality — pure dependency correlation, completely independent of any annotation — still recovers hidden function at 12× the random rate. Two independent evidence chains both confirm: the signal is real biology, not laundered annotation.

8 · What it means for the feature

One engine, two uses:

Applied to…Top predictions are…Role
a well-studied gene (GABRA5)functions it already has → grades the method✓ recovered
a dark / under-annotated genefunctions it does not yet have → new hypotheses· novel

Because the method recovers hidden truth 30× above chance on genes we know, its novel predictions on genes we don't know are trustworthy hypotheses for experimental follow-up — not asserted facts. Every prediction ships with its evidence trail: which neighbours support it, from which layers, at what confidence.

Honest caveats

  • No direct leakage: a gene's own annotations are never used to predict itself; a small second-order effect (a neighbour annotated by transfer from the target) remains, and is why we lean on the orthogonal co-essentiality layer.
  • RETICLE's own layers are still modest because the current screen matrix is almost all fitness (growth) screens — one axis. Using the full phenotype-diverse screen set is expected to strengthen this differentiated signal.
  • A high score is not truth. Predictions are framed as ranked hypotheses, always with the supporting evidence exposed.
